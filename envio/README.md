# Envio — Rootstream HyperIndex

**Envio HyperIndex** indexes **Rootstream** contract events on **Rootstock Testnet (chain id 31)** and serves a **GraphQL API** for dashboards and integrations (for example the **`../frontend/`** app via `NEXT_PUBLIC_ENVIO_GRAPHQL_URL`).

Deploy the contract first (see **`../contracts/README.md`**), then align **`config.yaml`** (or env overrides) with that address and deployment block.

## What gets indexed

Handlers subscribe to every Rootstream event used by the UI and analytics:

| Event | Handler role |
|-------|----------------|
| `StreamCreated` | Creates `Stream`, updates sender/recipient `User`, bumps `Analytics` |
| `StreamCancelled` | Marks stream inactive, decrements active stream count |
| `PaymentExecuted` | Appends `Payment`, updates stream totals / `lastExecuted`, user paid aggregates |
| `FundsDeposited` | Appends `Deposit`, updates user deposits + `derivedBalance` |
| `FundsWithdrawn` | Appends `Withdrawal`, updates user withdrawals + `derivedBalance` |

ABI: **`abis/Rootstream.json`** — keep in sync with the Solidity build from **`../contracts/`** (for example `forge build` and copy or regenerate if the interface changes).

## GraphQL entities

Defined in **`schema.graphql`** (Envio generates types and DB layout from this):

| Entity | Purpose |
|--------|---------|
| `Stream` | Per-stream state, aggregates (`totalPaid`, `executionCount`), links to payments |
| `Payment` | One row per `PaymentExecuted` |
| `User` | Per-address aggregates; `derivedBalance` from deposit / payment / withdrawal events |
| `Deposit` | Per `FundsDeposited` |
| `Withdrawal` | Per `FundsWithdrawn` |
| `Analytics` | Single global row (`id: "global"`) for protocol-wide totals |

Example queries live in **`queries.graphql`**.

### Totals vs on-chain balances

- **`Stream.totalPaid`** and **`User` payment totals** follow **`PaymentExecuted`** only.
- **`User.derivedBalance`** is updated as: **+** `FundsDeposited`, **−** sender share on **`PaymentExecuted`**, **−** **`FundsWithdrawn`**. It is **indexer-derived**, not a live chain read.
- **On-chain truth** for spendable prepaid balance remains **`Rootstream.balances(user)`** (see contracts).

If the indexer starts **after** some history, **`PaymentExecuted`** may create a **minimal** `Stream` row when `StreamCreated` was never ingested (see `EventHandlers.ts`).

## Layout

| Path | Purpose |
|------|---------|
| `config.yaml` | Network, RPC, contract address, event list, handler path |
| `schema.graphql` | Entity model exposed over GraphQL |
| `src/EventHandlers.ts` | Event → entity updates |
| `abis/Rootstream.json` | Contract ABI for decoding |
| `queries.graphql` | Sample GraphQL for local testing |
| `generated/` | **Codegen output** — do not hand-edit; regenerated by `envio codegen` |

## Configuration

### `config.yaml` highlights

- **`networks[0].id`**: `31` (Rootstock testnet).
- **`start_block`**: First block to scan — set to your **contract deployment block** (or earlier only if you know there are no prior deployments at the same address).
- **`contracts.Rootstream.address`**: Default in file; override with env (below).
- **`rpc`**: Uses **`ROOTSTOCK_TESTNET_RPC_URL`** with a fallback URL. **`initial_block_interval` / `interval_ceiling`** (2000) align with providers that cap **`eth_getLogs`** range.

### Environment variables

You can export these before **`envio dev`** / **`envio start`**:

```bash
export ROOTSTREAM_ADDRESS=0xYourDeployedRootstream
export ROOTSTOCK_TESTNET_RPC_URL=https://your-rootstock-testnet-rpc
```

`config.yaml` reads **`${ROOTSTREAM_ADDRESS:-…}`** and **`${ROOTSTOCK_TESTNET_RPC_URL:-…}`**.

## Requirements

- **Node.js** (LTS)
- **pnpm** — **`package.json`** scripts call `pnpm envio …`. Install: [pnpm.io](https://pnpm.io/installation).

If you only use **npm**, run the same flows with **`npx envio codegen`**, **`npx envio dev`**, **`npx envio start`** instead of the pnpm scripts below.

## Setup and local dev

```bash
cd envio
pnpm install
pnpm codegen    # generates `generated/` from schema + config
pnpm dev        # local indexer + GraphQL (URL printed in the terminal)
```

After **`codegen`**, if you change **`schema.graphql`** or **`config.yaml`**, run **`pnpm codegen`** again.

### Production-style run

```bash
pnpm start
```

Use this when you want the **`envio start`** flow (long-running sync without the dev UX); exact behavior follows [Envio’s CLI docs](https://docs.envio.dev) for your installed version.

## Frontend wiring

Point the frontend at the indexer’s GraphQL HTTP endpoint, for example:

```env
NEXT_PUBLIC_ENVIO_GRAPHQL_URL=http://127.0.0.1:8080/v1/graphql
```

The frontend defaults to that URL if unset. Confirm the path with whatever **`pnpm dev`** (or your hosted indexer) prints; Envio typically exposes a Hasura-compatible **`/v1/graphql`** endpoint.
